## Hidden Assumptions That May Be False

1. The relevant constraint vocabulary is known before planning starts.
2. A feasibility oracle can be queried many times inside search without changing the search objective.
3. A failed candidate plan is cheap enough to execute, simulate, or repair.
4. Constraint violations expose the whole constraint family instead of only one bad edge.
5. Training traces cover the deployment-time active constraint combinations.
6. The symbolic state abstraction already contains the variables that decide feasibility.
7. Geometric feasibility is local to a sampled state or edge, not an instance-level mode.
8. Task constraints are specified by a human rather than latent in the scene.
9. The same action schema remains legal across tools, payloads, surfaces, and humans.
10. Contact modes are either enumerated or discovered during ordinary trajectory search.
11. Robot perception can directly observe every constraint-relevant physical property.
12. A learned feasibility score can be used as a verifier without changing completeness claims.
13. Repairing one failed plan does not induce many near-duplicate failures.
14. The cost of invalid execution is comparable to a planning-time computation.
15. Constraint activation is independent across constraint families.
16. Dynamic or human-imposed constraints stay fixed long enough for replanning.
17. The planner can separate hard physical constraints from preferences after the fact.
18. Object affordances are static labels rather than task-conditioned constraints.
19. Motion and task planners share the same notion of feasibility.
20. Constraint uncertainty only matters at execution time, not in branch ordering.
21. The planner's branching factor is acceptable before knowing which actions are illegal.
22. A verifier call produces no information useful beyond the current candidate edge.
23. Failure data is abundant enough to learn all rare but severe active constraints.
24. Constraint discovery should be a byproduct of search rather than a first-class phase.
25. Latent constraints are independent of the robot's diagnostic actions.

## Candidate Directions From Broken Assumptions

- **Active constraint signatures before planning:** Make the binding constraint set an explicit latent variable inferred by cheap diagnostic probes and scene cues before graph or TAMP search begins.
- **Constraint-family repair rather than edge repair:** When a failure occurs, infer the violated constraint family and remove an entire equivalence class of actions, not a single plan fragment.
- **Assumption stress tests for learned feasibility:** Construct adversarial task families where edge-level feasibility classifiers are accurate but still waste search because they miss instance-level active sets.
- **Planner-native precondition discovery:** Learn which precondition variables must be measured, then schedule measurements as part of planning under physical probe costs.
- **Constraint activation certificates:** Attach a compact certificate to a scene stating which constraint families are active and which planner branches are therefore sound to remove.

## Pre-Decision Takeaway
The strongest opening is not another verifier, learned feasibility model, or repair module. The sweep repeatedly assumes the active constraint set is known, supplied, or discovered incidentally. That makes pre-planning active-set discovery a plausible central mechanism if the paper can show it changes search complexity and invalid-execution cost.
